# MMRM 매수 시나리오: MACD 하락영역 상승흐름

이 문서는 기존 **매수 시나리오 개선본**을 프로그램 구현 경험에 맞게 보강한 문서이다.

기존 문서의 핵심 전략은 유지한다. 다만 실제 코드로 검증 도구를 만들면서 명확히 정해야 했던 세부 기준을 추가한다.

용어 정의는 별도 문서인 [MMRM 주봉 차트 설정 및 용어 사전](mmrm-chart-setup-and-glossary.md)을 따른다.

---

## 1. 기본 아이디어

주가가 아직 쌀 때, 본격적으로 올라가기 전에 미리 매수하는 것을 목표로 한다.

이미 많이 오른 뒤에 따라가는 것이 아니라, **MACD 하락영역에서 MACD 하락흐름이 끝나고 MACD 상승흐름으로 바뀌는 초기 구간**을 추적해 매수지점을 찾는다.

이 전략에서 중요한 것은 단순히 MACD가 0 아래에 있는가가 아니다.

중요한 것은 다음이다.

> MACD 하락영역에서 시작된 MACD 상승흐름이 아직 유지되고 있는가.

---

## 2. 핵심 원칙

| 항목 | 원칙 |
|---|---|
| 기준 차트 | 주봉 차트 |
| 기본 판정 기준 | 마감 확정된 주봉 기준 |
| 개별 종목 검색 | 마감 확정된 과거 주봉만 사용 |
| 이번주 Top 100 스캐너 | 진행 중인 이번 주봉까지 포함 |
| 이번주 스캐너 결과 이름 | 확정 매수포인트가 아니라 이번주 매수후보 |
| 핵심 출발점 | MACD 하락영역에서 시작된 MACD 상승흐름시작 |
| 실제 관찰 시작 | 하락영역 상승흐름시작 발생 시점 |
| 매수 가능 상태 | MACD 상승흐름이 유지되는 상태 |
| 매수 확인 지표 | Momentum, RSI, MFI |

주의:

- 개별 종목 검색 결과는 과거에 이미 마감된 주봉 기준의 매수포인트이다.
- Top 100 스캐너 결과는 이번 주 진행 중인 주봉을 포함하므로 확정 매수포인트가 아니라 **이번주 매수후보**이다.
- 이번 주봉은 금요일 마감 전까지 값이 바뀔 수 있다.

---

## 3. 관찰 시작 지점

### 3-1. 기본 관찰 시작 기준

| 순서 | 내용 |
|---|---|
| 1 | MACD 하락영역에서 MACD 하락흐름이 진행 중인 상태를 확인한다. |
| 2 | 이후 MACD선이 Signal선 위로 올라서며 MACD 상승흐름시작이 발생한다. |
| 3 | 이 상승흐름이 하락영역에서 시작된 상승흐름인지 확인한다. |
| 4 | 인정되면 이때부터 매수 후보 관찰을 시작한다. |

### 3-2. 하락영역 상승흐름시작 인정 기준

MACD 상승흐름시작이 발생한 주봉에서 아래 둘 중 하나를 만족하면 **하락영역에서 시작된 MACD 상승흐름**으로 인정한다.

| 조건 | 인정 여부 |
|---|---|
| 상승흐름시작 주봉의 MACD선이 0 아래에 있음 | 인정 |
| 상승흐름시작 주봉의 MACD선은 0 위로 올라왔지만, 직전 주봉의 MACD선이 0 아래에 있었음 | 인정 |
| 상승흐름시작 주봉과 직전 주봉 모두 MACD선이 0 위에 있었음 | 인정하지 않음 |

핵심 표현:

> MACD 상승흐름시작이 발생한 주에 MACD가 이미 0선을 살짝 넘어왔더라도, 직전 주 MACD가 0 아래였다면 하락영역에서 시작된 상승흐름으로 인정한다.

이 기준이 필요한 이유:

- 차트에서는 하락영역에서 올라오던 흐름이 0선을 살짝 넘는 순간과 상승흐름시작이 거의 동시에 보일 수 있다.
- 코드에서는 이 경우를 명확히 정해야 한다.
- 사용자의 의도는 하락영역에서 출발한 상승흐름을 추적하는 것이므로, 직전 주 MACD가 하락영역이었다면 인정한다.

## 4. 매수지점

### 4-1. 매수지점 조건

관찰 시작 이후, 아래 조건이 모두 만족되는 **첫 주봉**을 매수지점으로 본다.

| 구분 | 조건 |
|---|---|
| MACD | MACD 상승흐름 유지 중 |
| Momentum | Momentum > 0 |
| RSI | RSI > 50 |
| MFI | MFI > 50 |

개별 종목 검색에서는 위 조건이 모두 만족되는 **첫 마감 주봉**을 매수포인트로 기록한다.

Top 100 스캐너에서는 이번 주 진행 중인 주봉까지 포함해 위 조건이 만족되는 경우를 **이번주 매수후보**로 기록한다.

### 4-2. 매수포인트 날짜

매수포인트 표에 표시하는 날짜는 **조건이 완성된 주봉의 날짜**이다.

| 항목 | 의미 |
|---|---|
| 매수포인트날짜 | Momentum, RSI, MFI 조건이 처음 동시에 만족된 주봉의 날짜 |
| 관찰시작일 | MACD 하락영역 상승흐름시작이 발생한 내부 추적용 날짜 |

주의:

- 관찰시작일은 매수포인트 날짜가 아니다.
- 관찰시작일은 프로그램 내부 계산에는 필요하지만, 사용자가 보는 매수포인트 표에는 표시하지 않는다.
- 사용자가 실제로 보고 싶은 날짜는 매수포인트날짜이다.

## 5. MACD 영역 허용 범위

매수 시점의 MACD 영역은 고정하지 않는다.

| 매수 시점 MACD 상태 | 허용 여부 |
|---|---|
| MACD 상승흐름이 유지 중이고 아직 MACD 하락영역에 있음 | 허용 |
| MACD 상승흐름이 유지 중이고 이미 MACD 상승영역에 진입함 | 허용 |
| MACD 하락흐름으로 바뀜 | 매수 불가 |

중요한 것은 **현재 MACD가 반드시 하락영역에 남아 있어야 한다는 것**이 아니다.

중요한 것은 다음이다.

> MACD 하락영역에서 시작된 MACD 상승흐름이 매수 시점까지 끊기지 않고 유지되고 있는가.

따라서 관찰이 시작된 뒤 MACD가 0 위로 올라와도, MACD 하락흐름시작이 나오기 전까지는 관찰을 계속한다.

---

## 6. 관찰 무효화 규칙

매수는 반드시 **MACD 상승흐름이 진행 중일 때만 가능**하다.

Momentum, RSI, MFI 조건이 모두 만족되기 전에 MACD가 다시 하락흐름으로 바뀌면 기존 관찰 흐름은 종료한다.

| 상황 | 처리 |
|---|---|
| MACD 상승흐름시작 이후 세 조건을 기다리는 중 | 관찰 유지 |
| 관찰 중 MACD가 상승영역으로 올라감 | 관찰 유지 |
| 세 조건 충족 전까지 MACD 상승흐름 유지 | 매수 후보 흐름 유지 |
| 세 조건 충족 전에 MACD 하락흐름시작 발생 | 기존 관찰 흐름 종료 |
| 기존 관찰 흐름 종료 후 | 다음 하락영역 상승흐름시작을 새로 기다림 |

핵심 표현:

> 조건이 완성되기 전에 MACD 하락흐름시작이 나오면 그 흐름에서는 매수하지 않는다. 다음 하락영역 상승흐름시작을 기다린다.

---

## 7. 한 흐름에서의 매수포인트 기록 기준

하나의 MACD 상승흐름 안에서는 조건이 처음 완성되는 주봉만 매수포인트로 기록한다.

| 상황 | 처리 |
|---|---|
| 관찰 시작 후 세 조건이 처음 동시에 만족됨 | 매수포인트 1회 기록 |
| 다음 주에도 세 조건이 계속 만족됨 | 같은 흐름에서는 추가 기록하지 않음 |
| 이후 MACD 하락흐름시작으로 흐름 종료 | 관찰 종료 |
| 이후 새로운 하락영역 상승흐름시작 발생 | 새 관찰 시작 가능 |

이 기준은 같은 상승흐름 안에서 매수포인트가 여러 번 중복 기록되는 것을 막기 위한 것이다.

## 9. 이번주 매수후보 스캐너 기준

Top 100 스캐너는 개별 종목 검색과 목적이 다르다.

개별 종목 검색은 과거 매수포인트 검증용이고, Top 100 스캐너는 금요일 주봉 마감 직전에 이번 주 후보를 찾기 위한 기능이다.

| 항목 | 개별 종목 검색 | Top 100 스캐너 |
|---|---|---|
| 목적 | 과거 매수포인트 검증 | 이번주 후보 찾기 |
| 진행 중인 주봉 | 제외 | 포함 |
| 결과 이름 | 매수포인트 | 이번주 매수후보 |
| 데이터 | 캐시 사용 가능 | 매번 새로 다운로드 |
| 결과 저장 | 자동으로 outputs 폴더 저장 | 사용자가 저장 버튼을 누를 때 다운로드 폴더 저장 |

주의:

- 이번주 매수후보는 아직 확정된 매수포인트가 아니다.
- 금요일 종가 전에는 Close, Momentum, MACD, RSI, MFI가 바뀔 수 있다.
- 따라서 스캐너 결과는 "이번 주 마감 때까지 조건이 유지되는지 확인할 후보"로 본다.

## 10. 경계값 처리

조건은 모두 엄격하게 초과 기준을 사용한다.

| 항목 | 매수 조건 만족 |
|---|---|
| Momentum | Momentum > 0 |
| RSI | RSI > 50 |
| MFI | MFI > 50 |
| MACD 상승흐름 | MACD선 > Signal선 |

아래 값은 조건 만족으로 보지 않는다.

| 항목 | 처리 |
|---|---|
| Momentum = 0 | 조건 미충족 |
| RSI = 50 | 조건 미충족 |
| MFI = 50 | 조건 미충족 |
| MACD선 = Signal선 | 상승흐름도 하락흐름도 아닌 중립 |

---

## 11. 매수지점 요약 공식

> **MACD 하락영역에서 시작된 MACD 상승흐름시작이 발생한 뒤, 그 MACD 상승흐름이 끊기지 않고 유지되는 동안 Momentum > 0, RSI > 50, MFI > 50이 처음 동시에 만족되는 주봉이 매수포인트이다.**

보강된 핵심:

> **상승흐름시작 주봉에서 MACD가 이미 0 위로 올라왔더라도 직전 주 MACD가 0 아래였다면 하락영역에서 시작된 상승흐름으로 인정한다. 관찰 시작 후에는 MACD가 상승영역으로 올라가도 MACD 하락흐름시작이 나오기 전까지 관찰을 계속한다.**

## 12. 아직 이 문서에서 다루지 않는 것

아래 내용은 매수지점 정의 이후에 별도 문서나 별도 섹션에서 정리한다.

- 매수 후 보유 기준
- 매도 조건
- 손절 조건
- 추가매수 조건
- 실패 시나리오
- 제외 조건
- 너무 늦은 매수 구간 판단
- 재진입 기준
- 매수 후 Momentum / RSI / MFI가 다시 기준선 아래로 내려갈 때의 처리
- 매수 후 MACD 상승흐름이 끊길 때의 처리
