# MMRM 패턴 연구 — 최종 단일 패키지

이 폴더 하나로 MMRM 신규·과거 기준봉을 해석하고, 연구 확률을 조회하며, 프로그램 컬럼 확장과 매주 예약 분석을 이어갈 수 있다. 세션 채팅이나 `mmrm-research-notes` 작업 폴더를 권위 자료로 사용하지 않는다.

## 1. 이 패키지의 목적

MMRM은 매수후보를 찾는다. 이 패키지는 그 후보들을 다시 비교해 다음 질문에 답한다.

- 기준봉 매수 후 8주 안에 바로 올라갈 가능성은 어느 정도였는가?
- 52주 뒤 종가수익률이 +20% 이상이었던 비율은 얼마인가?
- 52주 안에 한 번이라도 +20%에 도달했는가?
- 목표까지 얼마나 걸렸고 중간 최대하락폭은 어느 정도였는가?
- ATH·장기선·MACD·저점 형태를 고려하면 어떤 상승경로와 위험을 기대할 수 있는가?
- 여러 검색 종목 중 어떤 기준봉을 먼저 볼 것인가?

최종 사용처는 두 가지다.

1. MMRM 프로그램 검색 결과에 패턴·확률·위험·판정 컬럼을 추가한다.
2. 매주 검색된 신규 종목을 ChatGPT/Codex가 같은 규칙으로 분석한다.

## 2. 연구의 현재 수준

이 연구에 대한 가장 정확한 평가는 다음과 같다.

> **무의미한 패턴 맞추기는 아니지만, 수익 우위가 완전히 증명된 매매전략도 아니다. 현재는 의미 있는 탐색연구이자 MMRM 후보 선별 보조도구다.**

의미가 있는 이유:

- 2015-10-05까지의 자료로 다섯 패턴을 만들고 2015-10-12 이후 사건에 고정 적용했다.
- 기준봉 이후 결과를 보지 않고 사전 52주 차트만으로 패턴을 배정했다.
- 사건별 확률은 당시 이미 결과가 성숙한 과거 사건만 사용했다.
- 성공사례뿐 아니라 실패·지연 사례도 포함했다.
- 신규 사건을 같은 방식으로 분류할 고정 모델을 보관했다.
- 원형 연구 2,801건과 KIS 호환 2,789건의 기존 패턴을 재현했으며 불일치는 0건이었다.

아직 증명되지 않은 것:

- MMRM 자체가 임의 주봉이나 단순 기술지표보다 우월한가
- 시장지수·섹터지수보다 초과수익을 내는가
- 현재 패턴 순위가 완전히 새로운 미래자료에서도 유지되는가
- 실제 투자에서 연복리 20%를 달성하는가
- 프로그램의 `매수` 판정이 통계적으로 최적인가

따라서 패턴 확률은 후보 간 비교와 위험 설명에 사용하고, 확정적인 미래예언으로 사용하지 않는다.

## 3. 이 패키지로 할 수 있는 일

### 3.1 한 기준봉 분석

- MMRM 공식 기준봉 여부와 계산 엔진 확인
- 다섯 패턴 중 하나로 분류
- 패턴 중심과의 거리 및 경계·이상치 여부 확인
- 8·26·52주 상승빈도 확인
- 52주 뒤 +20%, 52주 내 +20%, +20% 소요기간 확인
- 중앙수익률과 최대하락폭 확인
- ATH 안전마진과 이전 ATH 회복 필요수익률 계산
- 150·200주선, MACD, V/U 저점, 지지선, TFMR 경고 해석
- 기준봉 당시 정보만으로 `매수` 또는 `매수하지 않음` 판정
- 투자 논리가 성립하는 기대와 반대로 조심할 점 설명

### 3.2 여러 검색 종목 비교

- 52주 +20% 확률이 높은 순서
- 8주 즉각성이 좋은 순서
- 최대하락폭이 덜 나쁜 순서
- +20% 도달기간이 짧은 순서
- 패턴 중심에 가까운 순서
- ATH·장기선·TFMR 위험을 반영한 최종 우선순위

### 3.3 성공·실패 경로 비교

같은 패턴의 과거 사건에서 다음을 함께 비교할 수 있다.

- 강한 즉시 상승·장기 확장
- 완만한 즉시 상승 지속
- 초기 하락 후 느린 회복
- 초기 버팀 후 중장기 붕괴
- 즉시 급락·장기 부진

이후 흐름은 과거 결과를 설명하는 사후 연구이며 신규 기준봉에 이미 발생한 사실처럼 붙이지 않는다.

### 3.4 프로그램 및 예약 분석 확장

- 프로그램 결과표에 패턴·거리·확률·MAE·ATH·경고·판정 컬럼 추가
- 매주 예약 분석에서 검색 종목 순위와 쉬운 해설 제공
- 대형 상승 가능성이 있는 종목을 `대형 상승 탐색후보`로 별도 표시
- 새로운 결과가 성숙하면 동일 구조로 통계 갱신

`대형 상승 탐색후보`는 확정적인 대박주나 검증된 별도 성공등급을 뜻하지 않는다.

## 4. 할 수 없는 일과 한계

### 4.1 미래수익 보장

높은 역사확률도 개별 종목의 상승·+20% 달성·손실 없는 상승을 보장하지 않는다. 확률은 과거 유사사례 집단의 빈도다.

### 4.2 티커만으로 자동 분석

신규 기준봉 분석에는 기준봉 이전 52주 주봉, OHLCV, 보조지표, 이동평균선, ATH 구조가 필요하다. 이 패키지만으로 최신 시세가 자동 생성되지는 않는다.

### 4.3 기업가치와 외부환경 판단

현재 연구는 차트·기술지표 중심이다. 실적, 밸류에이션, 산업 경쟁력, 금리, 규제, 소송, 뉴스, 시장충격은 공식 확률 입력이 아니다.

### 4.4 완전 자동화되지 않은 판단

- V/U 저점과 수평 지지선은 수동 해석이 남아 있다.
- 리버스 TFMR는 공식 TFMR이 아니라 사용자 사례로 복원한 별도 가설이다.
- 손절, 분할매수, 목표가 매도, 자산배분은 별도 전략이 필요하다.
- 수수료·세금·환율·슬리피지·실제 다음 주 체결가는 반영하지 않았다.

### 4.5 연구 데이터의 구조적 한계

- 현재 시가총액 상위 100개 생존 종목을 과거로 거슬러 사용한 생존편향이 있다.
- 동일 종목의 반복 사건은 완전히 독립적이지 않다.
- 52주 결과구간이 서로 겹치는 사건이 있다.
- `investing_proxy`와 `kis_compatible`은 공급자 공식 전체 원천을 완전히 복제한 자료가 아니다.
- 패턴·ATH·TFMR 등 여러 가설을 탐색했으므로 우연히 좋아 보이는 조건이 선택됐을 가능성이 있다.

## 5. 패턴 맞추기 환각을 줄인 장치

- 패턴 생성구간과 이후 평가구간을 시간순으로 분리
- 신규 사건에 고정 중심점을 사용하고 재학습하지 않음
- 패턴의 사람용 이름과 실제 숫자 모델을 분리
- 기준봉 이후 결과를 사전 패턴 입력에서 제외
- 성공·실패·지연 사례를 모두 보존
- 표본 30건 미만 패턴은 공식 순위에서 제외
- 패턴 거리 95백분위 밖은 `outlier`로 표시
- Investing 근사판과 KIS 호환판을 섞지 않음
- 검증 스크립트와 MSFT 회귀검사 사례 보관

이 장치들이 환각 위험을 줄여주지만 생존편향·다중검정·시장환경 변화까지 없애지는 못한다.

## 6. 올바른 사용법

적합한 사용:

- MMRM에 잡힌 후보 중 상대적으로 더 나은 기준봉을 고른다.
- 사용자의 1년 +20% 목표와 맞지 않는 패턴을 거른다.
- 상승 기대와 실패·지연 위험을 함께 이해한다.
- 매주 같은 기준으로 일관되게 판정한다.
- 실제 미래 신호를 사전 기록하고 나중에 검증한다.

금지되는 사용:

- `확률 65%이므로 반드시 오른다`고 단정
- `매수` 판정만 보고 큰 금액을 집중투자
- `대형 상승 탐색후보`를 확정적인 인생 역전 종목으로 표현
- 실패할 때마다 조건과 패턴을 바꿔 과거 결과에 다시 맞춤
- 사후 수익을 본 뒤 당시 패턴과 매수판정을 변경

## 7. 앞으로 연구가 통과해야 할 검증

현재 패키지는 앞으로 다음 검증의 기준점이다.

1. 현재 패턴·특징·운영선을 고정한다.
2. 매주 실제 검색 신호와 당시 판정을 결과가 나오기 전에 저장한다.
3. 8·26·52주 뒤 매수군과 비매수군을 비교한다.
4. 각 과거 시점의 당시 시가총액 상위 종목을 복원해 생존편향을 줄인다.
5. 비슷한 낙폭·장기선·시장환경의 비MMRM 주봉과 대조한다.
6. SPY·QQQ·섹터지수 대비 초과수익을 확인한다.
7. 같은 종목 반복과 겹치는 결과기간을 고려한 종목·시간 블록 검증을 한다.

이 검증에서도 우위가 유지돼야 `탐색연구`에서 `실전 검증전략`으로 높일 수 있다.

## 10. 공식 기준과 매수 운영정책

- 기본 권위 엔진: `investing_proxy`
- 보조 민감도 엔진: `kis_compatible`
- 패턴 분류체계 학습 종료: `2015-10-05`
- 워크포워드 평가 시작: `2015-10-12`
- 최소 공식 확률 표본: 30건
- 기본 투자 목표: 기준봉 종가 매수 후 52주 종가수익률 `+20% 이상`
- 기본 운영 판정선: 52주 후 +20% 역사확률 `40%`

사용자가 별도 기준을 말하지 않으면 다음을 모두 충족할 때 기본 `매수`로 판정한다.

1. 공식 MMRM 기준봉
2. 공식확률 표본 30건 이상
3. 52주 뒤 +20% 역사확률 40% 이상
4. 패턴 거리 상태가 `outlier`가 아님
5. 심각한 데이터 품질 경고가 없음

하나라도 충족하지 못하면 기본 판정은 `매수하지 않음`이다.

`40%`는 연구가 증명한 자연법칙이나 최적값이 아니라 세션마다 판정이 바뀌지 않도록 둔 사용자용 운영선이다.

## 11. 신규 기준봉 처리

- 과거 사건이 `data/events_investing_proxy.csv`에 있으면 해당 행과 `data/walk_forward_predictions.csv`를 사용한다.
- 새 사건이면 기준봉 이전 52주 주봉과 기준봉 구조값을 `tools/pattern_classifier.py`에 넣어 패턴을 고정 분류한다.
- 새 사건의 확률은 `data/pattern_metrics.csv`의 동일 패턴 공식값을 사용한다.
- 기준봉 뒤 결과는 매수판정에 사용하지 않는다. 사후검산은 판정과 분리한다.
- 데이터가 없으면 추측하지 않고 `분석자료 부족`이라고 명시한다.

## 12. 프로그램 연동

`tools/pattern_classifier.py`의 `classify_event()`를 불러오면 다음 값을 얻는다.

- `pattern_id`, `pattern_name`
- `pattern_distance`
- `distance_status`, `distance_label`
- `training_cluster_n`

입력 열과 프로그램 출력 열의 전체 규격은 `MMRM_KNOWLEDGE_BASE.md` 13장에 있다.

## 14. 검증
