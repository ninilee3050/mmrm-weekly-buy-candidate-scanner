# MMRM 주봉 차트 설정 및 용어 사전

이 문서는 기존 **주봉 차트 화면 세팅 및 소통용 용어 사전**을 프로그램 구현 경험에 맞게 보강한 문서이다.

이 문서의 역할은 다음과 같다.

1. 사용자가 보는 차트 화면 세팅을 기록한다.
2. 화면에 보이는 선, 색상, 기준선을 같은 이름으로 부르기 위한 용어를 정의한다.
3. 사람과 코드가 같은 기준으로 지표를 계산하도록 계산 기준을 명확히 한다.
4. 차트 서비스마다 값이 다를 수 있는 부분을 기록해 오해를 줄인다.

이 문서는 매수 조건 자체를 판단하는 문서가 아니다. 매수 조건은 [MMRM 매수 시나리오](mmrm-buy-scenario.md)에서 다룬다.

## 1. 화면 세팅

### 1-1. 차트 기본

| 항목 | 세팅 |
|---|---|
| 차트 종류 | 캔들 차트 |
| 기준 주기 | 주봉 |
| 기본 판정 기준 | 마감된 주봉 기준 |
| 개별 종목 검색 | 진행 중인 주봉 제외 |
| 이번주 스캐너 | 진행 중인 주봉 포함 가능 |

주의:

- 기본 용어 판정은 마감 확정된 주봉 기준이다.
- 진행 중인 주봉은 값이 바뀔 수 있으므로 확정된 과거 판단에는 사용하지 않는다.
- 다만 Top 100 스캐너는 금요일 마감 직전 후보를 찾기 위한 기능이므로 진행 중인 이번 주봉을 포함할 수 있다.
- 이 경우 결과는 확정 매수포인트가 아니라 **이번주 매수후보**라고 부른다.

## 2. 주봉 데이터 기준

프로그램은 Yahoo Finance의 원본 `1wk` 주봉을 그대로 사용하지 않는다.

실제 구현 과정에서 Yahoo의 `1wk` 주봉 라벨과 Investing.com 주봉 화면이 다르게 보이는 사례가 있었다. 예를 들어 어떤 종목은 Investing.com에서는 월요일 시작 주봉처럼 보이지만 Yahoo `1wk` 원본은 목요일 라벨로 나오는 경우가 있었다.

그래서 프로그램은 다음 방식을 사용한다.

| 항목 | 기준 |
|---|---|
| 원천 데이터 | Yahoo Finance 일봉 데이터 |
| 1차 다운로드 | Yahoo chart API |
| 보조 다운로드 | yfinance |
| 주봉 생성 방식 | 일봉을 월요일 시작 주봉으로 직접 재구성 |
| 주봉 날짜 라벨 | 해당 주의 월요일 |
| 개별 검색 현재 주봉 | 제외 |
| Top 100 스캐너 현재 주봉 | 포함 |

### 2-1. 일봉을 주봉으로 묶는 기준

| 주봉 항목 | 계산 기준 |
|---|---|
| Open | 해당 주 첫 거래일 시가 |
| High | 해당 주 최고가 |
| Low | 해당 주 최저가 |
| Close | 해당 주 마지막 거래일 종가 |
| Volume | 해당 주 거래량 합계 |
| Adj Close | 해당 주 마지막 거래일 수정종가 |

주의:

- `Date` 또는 `매수포인트날짜`는 실제 매수 주문 시간이 아니다.
- 프로그램에서 표시하는 날짜는 해당 주봉의 시작일 라벨이다.
- 현재 기준에서는 월요일 시작 주봉으로 표시한다.

## 3. 가격 보정 기준

주식은 분할, 병합, 배당 등의 이유로 과거 가격이 보정될 수 있다.

프로그램은 Yahoo에서 제공하는 OHLCV 데이터를 기준으로 계산한다.

| 항목 | 현재 기준 |
|---|---|
| 지표 계산용 가격 | Close |
| 수정종가 저장 | Adj Close가 있으면 함께 저장 |
| 지표 계산에 Adj Close 사용 여부 | 사용하지 않음 |

주의:

- Yahoo, Investing.com, 모바일 차트 앱은 분할, 배당, 가격 보정 방식이 다를 수 있다.
- 따라서 같은 종목, 같은 날짜라도 지표값이 완전히 같지 않을 수 있다.
- 이 프로그램은 하나의 고정된 계산 기준을 사용해 매수 시나리오를 일관되게 검증하는 도구이다.

## 6. 보조지표 세팅

| 지표 | 세팅 | 계산 기준 | 화면 기준 |
|---|---|---|---|
| Momentum | 14주, Close 기준 | Close - 14주 전 Close | 0선 기준 |
| MACD | 12, 26, 9, Close 기준 | EMA 12, EMA 26, Signal EMA 9 | 0선, MACD선, Signal선 |
| RSI | 14주 | Wilder 방식 | 50선, 30선, 70선 |
| MFI | 14주 | Typical Price와 거래량 기준 | 50선, 30선, 70선 |

### 6-1. Momentum 계산

```text
Momentum = 현재 Close - 14주 전 Close
```

### 6-2. MACD 계산

```text
EMA12 = Close 기준 12기간 EMA
EMA26 = Close 기준 26기간 EMA
MACD선 = EMA12 - EMA26
Signal선 = MACD선 기준 9기간 EMA
히스토그램 = MACD선 - Signal선
```

중요:

- MACD Signal선은 표준 방식인 EMA 9를 사용한다.
- 한때 PC Investing.com 화면과 맞추기 위해 Signal선을 SMA로 볼 수 있는지 검토했지만, 모바일 앱과 표준 계산식 차이를 확인한 뒤 표준 EMA 방식으로 유지한다.
- 차트 서비스마다 MACD Signal 계산 방식이나 초기값 처리 방식이 달라 보일 수 있다.

### 6-3. RSI 계산

RSI는 14기간 Wilder 방식을 사용한다.

주의:

- 일부 차트 서비스는 초기값 처리나 보정 방식 차이로 RSI 값이 다르게 보일 수 있다.
- 프로그램은 같은 데이터와 같은 계산식으로 일관되게 계산하는 것을 우선한다.

### 6-4. MFI 계산

```text
Typical Price = (High + Low + Close) / 3
Money Flow = Typical Price * Volume
MFI = 14기간 양의 Money Flow와 음의 Money Flow 비율 기준
```

주의:

- MFI는 가격뿐 아니라 거래량도 사용한다.
- 데이터 제공처의 거래량 차이가 있으면 MFI 값도 달라질 수 있다.

## 8. Momentum 용어

Momentum은 0선을 기준으로 위쪽과 아래쪽 영역을 나누어 부른다.

| 기준 | 사용자 정의 용어 |
|---|---|
| Momentum 값이 0선 초과 | Momentum 상승영역 |
| Momentum 값이 0선 미만 | Momentum 하락영역 |
| Momentum 값이 0 | 기준선 |

매수 시나리오에서 Momentum 조건은 `Momentum > 0`이다.

## 9. MACD 용어

MACD는 다음 다섯 가지 개념을 분리해서 본다.

| 구분 | 의미 |
|---|---|
| 영역 | 0선을 기준으로 나눈 위쪽 / 아래쪽 차트 공간 |
| 흐름 | MACD선과 Signal선의 위아래 관계 |
| 흐름시작 | MACD선과 Signal선의 우위가 바뀌는 순간 |
| 영역진입 | MACD선이 0선을 넘어 상승영역 또는 하락영역으로 들어가는 순간 |
| 발산 / 수렴 | MACD선과 Signal선 사이의 간격 변화 |

### 9-1. MACD 화면 선 이름

| 화면 표현 | 실제 용어 | 이 문서에서 사용할 이름 |
|---|---|---|
| 빨간색 선 | MACD선 | MACD선 |
| 파란색 선 | Signal선 | Signal선 |
| 연한 막대 | 히스토그램 | 히스토그램 |

### 9-2. MACD 영역 용어

| 기준 | 사용자 정의 용어 |
|---|---|
| 0선 위쪽 차트 공간 | MACD 상승영역 |
| 0선 아래쪽 차트 공간 | MACD 하락영역 |

주의:

- 영역은 차트 공간 이름이다.
- 두 선이 동시에 같은 영역에 있어야 한다는 조건을 뜻하지 않는다.
- 매수 시나리오에서 하락영역 여부는 MACD선을 기준으로 판단한다.

### 9-3. MACD 흐름 용어

| 기준 | 사용자 정의 용어 |
|---|---|
| MACD선 > Signal선 | MACD 상승흐름 |
| Signal선 > MACD선 | MACD 하락흐름 |
| MACD선 = Signal선 | 교차 지점 / 중립 |

### 9-4. MACD 흐름시작 용어

| 기준 | 사용자 정의 용어 |
|---|---|
| 전 주봉에서 MACD선 > Signal선, 이번 주봉에서 Signal선 > MACD선 | MACD 하락흐름시작 |
| 전 주봉에서 Signal선 > MACD선, 이번 주봉에서 MACD선 > Signal선 | MACD 상승흐름시작 |

주의:

- 흐름시작은 전 주봉과 이번 주봉의 관계를 비교해 판단한다.
- MACD선과 Signal선이 같은 값인 주봉은 상승흐름도 하락흐름도 아닌 중립으로 본다.

### 9-5. 하락영역 상승흐름시작

매수 시나리오에서 특히 중요한 용어이다.

| 기준 | 처리 |
|---|---|
| MACD 상승흐름시작 발생 주봉의 MACD선이 0 아래 | 하락영역 상승흐름시작 인정 |
| MACD 상승흐름시작 발생 주봉의 MACD선은 0 위지만, 직전 주봉 MACD선이 0 아래 | 하락영역 상승흐름시작 인정 |
| 현재 주봉과 직전 주봉 모두 MACD선이 0 위 | 하락영역 상승흐름시작으로 보지 않음 |

이 용어는 단순한 화면 용어를 넘어 매수 시나리오에서 관찰 시작 여부를 정하는 기준이다.

### 9-6. MACD 영역진입 용어

영역진입은 Signal선이 아니라 **MACD선 기준**으로 부른다.

| 기준 | 사용자 정의 용어 |
|---|---|
| MACD선이 0선을 위에서 아래로 넘어가는 순간 | MACD 하락영역진입 |
| MACD선이 0선을 아래에서 위로 넘어가는 순간 | MACD 상승영역진입 |

주의:

- 여기서 진입은 매수 진입을 뜻하지 않는다.
- 여기서 진입은 MACD선이 0선을 넘어 특정 영역으로 이동하는 순간을 뜻한다.

### 9-7. MACD 발산 / 수렴 용어

| 기준 | 사용자 정의 용어 |
|---|---|
| 전 주봉 대비 MACD선과 Signal선 사이의 간격이 커지는 상태 | MACD 발산 |
| 전 주봉 대비 MACD선과 Signal선 사이의 간격이 작아지는 상태 | MACD 수렴 |

## 10. RSI / MFI 공통 용어

RSI와 MFI는 동일한 용어 체계를 사용한다.

| 기준 | 사용자 정의 용어 |
|---|---|
| RSI 또는 MFI 값이 50선 초과 | RSI / MFI 상승영역 |
| RSI 또는 MFI 값이 50선 미만 | RSI / MFI 하락영역 |
| RSI 또는 MFI 값이 30선 이하 | 침체 |
| RSI 또는 MFI 값이 70선 이상 | 과열 |
| RSI 또는 MFI 값이 50 | 기준선 |

매수 시나리오에서는 아래 조건을 사용한다.

| 지표 | 조건 |
|---|---|
| RSI | RSI > 50 |
| MFI | MFI > 50 |

주의:

- 실제 소통에서는 RSI와 MFI를 구분해서 표현한다.
- 예: RSI 상승영역, MFI 하락영역, RSI 침체, MFI 과열.

## 11. 경계값 처리

경계값은 상승 / 하락 중 하나로 억지로 넣지 않고, 기준선 또는 중립 상태로 둔다.

| 상황 | 소통용 표현 | 매수 조건 만족 여부 |
|---|---|---|
| Momentum = 0 | 기준선 | 미충족 |
| MACD선 = 0 | 기준선 | 하락영역 아님 |
| MACD선 = Signal선 | 교차 지점 / 중립 | 상승흐름 아님 |
| RSI = 50 | 기준선 | 미충족 |
| MFI = 50 | 기준선 | 미충족 |
| RSI 또는 MFI = 30 | 침체에 포함 | 매수 조건과 무관 |
| RSI 또는 MFI = 70 | 과열에 포함 | 매수 조건과 무관 |

## 12. 차트 서비스 차이 주의

프로그램을 만들면서 다음 차이를 확인했다.

| 차이 발생 지점 | 설명 |
|---|---|
| Yahoo `1wk` 주봉 | 날짜 라벨이 Investing.com과 다르게 나올 수 있다. |
| PC Investing.com | 일부 지표값이 다른 앱과 다르게 보일 수 있다. |
| 모바일 차트 앱 | MACD Signal, RSI, MFI 값이 PC 화면과 다를 수 있다. |
| 데이터 보정 | 분할, 배당, 원천 데이터 차이로 과거 가격이 다를 수 있다. |
| 지표 초기값 | EMA, RSI, MFI 초기 계산 방식에 따라 값이 달라질 수 있다. |

따라서 이 프로그램의 기준은 다음처럼 고정한다.

1. Yahoo 일봉 데이터를 받는다.
2. 월요일 시작 주봉으로 직접 재구성한다.
3. Momentum, MACD, RSI, MFI 계산식을 명시한 방식으로 고정한다.
4. 개별 검색은 마감 주봉만 사용한다.
5. Top 100 스캐너는 이번주 후보 확인용으로 진행 중인 주봉을 포함한다.

이 기준은 특정 차트 앱의 모든 숫자를 100% 복제하기 위한 것이 아니라, 하나의 일관된 기준으로 전략 조건을 검증하기 위한 것이다.

## 13. 전체 요약

화면 세팅은 다음과 같다.

**주봉 캔들 + 이동평균선 5 / 20 / 50 / 150 / 200 + 거래량 50주 평균선 + Momentum 14주 + MACD 12,26,9 + RSI 14주 + MFI 14주**

계산 기준은 다음과 같다.

| 지표 | 계산 기준 |
|---|---|
| Momentum | Close - 14주 전 Close |
| MACD | Close 기준 EMA12 - EMA26, Signal은 MACD 기준 EMA9 |
| RSI | 14기간 Wilder 방식 |
| MFI | 14기간 Typical Price와 거래량 기준 |

가장 중요한 원칙은 다음과 같다.

1. 이 문서는 소통용 용어와 계산 기준을 맞추기 위한 문서이다.
2. 매수 판단 자체는 별도 매수 시나리오 문서에서 다룬다.
3. 개별 종목 검색은 마감 확정된 주봉 기준이다.
4. Top 100 스캐너는 진행 중인 이번 주봉을 포함할 수 있으며, 이 결과는 이번주 매수후보라고 부른다.
5. Yahoo 원본 `1wk` 주봉은 사용하지 않고 일봉을 월요일 시작 주봉으로 직접 묶는다.
6. 차트 서비스마다 값이 다를 수 있으므로, 프로그램은 명시된 계산 기준을 따른다.
